## Workflow: Plan > Execute > Review > Document

all non-trivial work follows this pipeline. do not skip steps.

1. **plan**: `/plan [task]` spawns @orchestrator (opus, plan mode). it reads the codebase, draws file boundaries, and produces a plan with a file ownership map. NO code is written. wait for user approval.
2. **execute**: `/ship` spawns @dev agents in parallel isolated worktrees. each agent only touches its assigned files. after all complete, branches merge back to main.
3. **review**: @reviewer (opus) audits the combined diff from all merged branches. checks for bugs, merge artifacts, and cross-agent inconsistencies.
4. **document**: @architect (opus) updates DOC-MAP.md, ARCHITECTURE.md, and any affected docs.

## Worktree Rules

- dev agents run in isolated worktrees via `isolation: worktree`
- the orchestrator's file ownership map determines what each agent can touch
- NO two parallel agents touch the same file. ever.
- shared files (package.json, tsconfig, types/index.ts, config files) are handled in a sequential wave AFTER parallel work merges
- add `.claude/worktrees/` to .gitignore

## Sub-Agent Routing

**parallel dispatch** (each agent gets its own worktree):
- 2+ independent tasks with zero file overlap
- clear file ownership per agent
- no shared state

**sequential dispatch** (same branch, one after another):
- tasks that touch the same files
- shared config/type updates after parallel merge
- review and documentation (always sequential, always last)

## Context Management

- manual /compact at 50% context usage
- /clear between unrelated tasks
- subagents return summaries, not raw output
- commit early and often
- write findings to markdown before compacting
